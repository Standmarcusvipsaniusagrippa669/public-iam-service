# Guía de Contribuciones - Valiantech IAM Core API

## Cómo Contribuir

Gracias por tu interés en contribuir al proyecto Valiantech IAM Core API. Para mantener la calidad y orden del proyecto, sigue estas indicaciones:

---

## Proceso para Contribuir

1. **Fork del repositorio:** Haz un fork para trabajar en tu propia copia.  
2. **Crea una rama:** Trabaja en ramas con nombre descriptivo, ej: `feature/login-2fa` o `fix/token-refresh`.  
3. **Desarrolla y prueba:** Asegúrate de escribir tests unitarios y que todas las pruebas pasen.  
4. **Documenta:** Actualiza documentación si agregas o cambias funcionalidades.  
5. **Pull Request:** Envía tu PR al repositorio principal describiendo claramente los cambios.

---

## Estándares de Código

- Sigue las convenciones de Java y Spring Boot.  
- Usa nombres descriptivos y claros para clases, métodos y variables.  
- Escribe comentarios y JavaDoc donde sea relevante.  
- Mantén el código limpio y legible.

---

## Tests

- Cubre funcionalidades nuevas y modificadas con tests unitarios.  
- Usa Mockito y JUnit 5 según la convención del proyecto.  
- No rompas tests existentes sin justificación.

---

## Revisión y Aprobación

- Los PR serán revisados por el equipo principal.  
- Pueden solicitar cambios o mejoras.  
- Una vez aprobado, se hará merge a la rama principal.

---

## Reporte de Bugs y Solicitudes de Funcionalidad

- Usa el sistema de issues para reportar bugs o proponer mejoras.  
- Proporciona detalles claros y pasos para reproducir problemas.

---
